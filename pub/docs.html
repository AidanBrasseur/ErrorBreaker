Getting Started

Using ErrorBreaker is very simple, but first we need to set up a few dependencies before we can get started.

ErrorBreaker utilizes jQuery, and thus you must include jQuery as a script in the head of your HTML as follows 

                
                    <script defer src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>

ErrorBreaker also utilizes a custom font that must be imported. Include this font in the head of your HTML as follows 

                
                    <link href='https://fonts.googleapis.com/css?family=Black+Ops+One' rel='stylesheet' type='text/css'>

Finally, we are ready to include the main ErrorBreaker.js library code. This code can be found here. Include this script in the head of your HTML as follows 

                
                    <script defer type="text/javascript" src='ErrorBreaker.js'> </script>

Now that we're all set up, we can begin creating our first game with ErrorBreaker!

Create an HTML div element and ensure that you set its width and height.

Next, in your JavaScript code, create an instance of the ErrorBreaker object, passing in a single object with this div and all other required attributes. Creating this object will spawn the ErrorBreaker game in the div provided.

Creating the ErrorBreaker Game

To create the ErrorBreaker object, pass in a single object with the following attributes to the constructor

Object Attribute Type Description
levelText string or number An alphanumeric text which dictates what the bricks in the level will be shaped as
message string [optional] The message to display above the game. Typically used to provide information about an error
div An HTML div element The HTML div element in which the game will be rendered
primaryColour CSS recognized colour string or hexadecimal colour code The primary colour of the bricks. The bricks that compose the levelText will be this colour
secondaryColour CSS recognized colour string or hexadecimal colour code The secondary colour of the bricks. The bricks that surround the levelText will be this colour
ballColour CSS recognized colour string or hexadecimal colour code The colour of the ball
messageColour CSS recognized colour string or hexadecimal colour code [required if message attribute is specified] The colour of the optional message text displayed above the game
powerupColour CSS recognized colour string or hexadecimal colour code The colour of the powerup icons that appear during the game
loading boolean [optional] A boolean to specify if the built in progress bar should be displayed
progressColour CSS recognized colour string or hexadecimal colour code [required if loading attribute is true] The colour of the loading bar fill
onClose function [optional] A function that will be called when the ErrorBreaker game is closed by user input or by a developer invoking the animateClose method
collisionSound src file string [optional] A path or url to a valid audio file. This audio will be played when the ball collides with an obstacle
secondaryCollisionSound src file string [optional] A path or url to a valid audio file. This audio will be played when the ball collides with an obstacle. Alternates with collisionSound
gameOverSound src file string [optional] A path or url to a valid audio file. This audio will be played when all balls have been lost.
winSound src file string [optional] A path or url to a valid audio file. This audio will be played when all bricks have been cleared.
powerupSound src file string [optional] A path or url to a valid audio file. This audio will be played when the player collects a powerup.
slam boolean [optional] A boolean to specify whether the levelText should initially animate in with a slam animation.

Interacting with the Game

The majority of the functionality of ErrorBreaker occurs during construction, but there are a few methods that are exposed to the developer after the object is made.

setProgress

Set the progress of the loading bar. loading and progressColour attributes must be specified during construction in order to use this method.

Parameters Type Description
percentage number Sets the loading bar above the game to be filled up to the given percentage. If percentages reach or exceed 100, the player will have the option to dismiss the game with an on screen button.
